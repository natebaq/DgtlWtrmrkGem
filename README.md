# 🛡️ Digital Watermarker (디지털 워터마커)

> **폴더 구조를 완벽히 보존하는 배치(Batch) 멀티 워터마킹, 눈에 보이지 않는 스테가노그래피(Steganography) 서명, 그리고 SHA-256 해시 검증을 지원하는 전문 창작자용 저작권 보호 솔루션**

---

## 1. 만들어진 의도와 목적

콘텐츠 디자이너, 사진작가, 일러스트레이터, 영상 제작자 등 수많은 크리에이터가 마주하는 가장 큰 고민은 **"저작물 도용"**과 **"아카이빙의 번거로움"**입니다. 

기존의 일반적인 워터마크 툴들은 여러 장의 이미지를 처리할 때 파일을 일일이 꺼내서 수동 작업해야 했으며, 작업 결과물을 저장하는 과정에서 고 수고스럽게 분류해 둔 **정밀한 폴더링과 네이밍 체계가 전부 깨져버리는 한계**가 있었습니다. 또한, 단순한 시각적 워터마크는 크롭(자르기)이나 지우개 도구로 손쉽게 제거되기 때문에 실질적인 **법적 보호 장치로 기능하기 어려웠습니다.**

**Digital Watermarker**는 현업 디자이너의 작업 흐름(Workflow)을 완벽히 존중하면서도, 기술적·사법적 저작권 효력을 더하기 위해 탄생했습니다:
1. **아카이빙 흐름 존중**: 수백 개의 폴더 구조를 지닌 마스터 폴더를 통째로 올리더라도, **원형 그대로의 폴더 트리 및 파일 물리 경로를 기억했다가 고스란히 재조립**해 압축 파일로 돌려줍니다.
2. **이중 장벽 보호 (Hybrid Protection)**: 눈에 보이는 시각적 마크와 함께 픽셀에 암호로 연동되는 **보이지 않는 '스테고 씰(Stego Seal)'**을 삽입하여 도용 시 원본 입증을 극대화합니다.
3. **사법 증거력 확보 (SHA-256 Digital Fingerprint)**: 변환 전후의 파일 데이터에서 디지털 지문 수준의 고유 해시(SHA-256)를 즉각 실시간 연산하고, 이를 **공식 CSV 리포트 Excel 대장**으로 영구히 보관할 수 있게 하여 법정 무결성 증빙 자료를 확보합니다.

## 2. 앱의 주요 기능

### 📁 1. 다중 폴더 트리 무한 재조립 (Recursive Directory Asset-Mapping)
- 드래그 앤 드롭 또는 폴더 선택만으로 하위 폴더가 몇 단계(`Depth`) 깊든 제한 없이 내부에 들어있는 전체 타겟 파일을 실시간으로 읽어들입니다.
- 개별 배치 처리가 끝난 뒤에는 **`[JSON / ZIP 아카이브 수집기]`** 장치가 구동되어 로컬 PC에 보존되어 있던 모든 깊이의 하위 폴더 계층을 완벽히 재구성한 단일 구조화 `.zip` 파일로 즉시 다운로드 가능합니다.

### 🚫 2. 지능형 어도비 확장자 필터링 (Adobe Format Automagic Filter)
- 아카이빙 폴더 내부에 섞여있는 작업용 고용량 확장자(`.psd`, `.ai`, `.eps`, `.indd`, `.pdf`)를 스캔 과정에서 자동으로 탐색하고 스킵(Exclusion)합니다.
- 변환 작업 중 무거운 어도비 소스 파일이 포함되면서 발생하는 렌더링 지연과 시스템 과부하를 즉각적으로 차단하고 오직 표준 웹 미디어 최적화 파일만 큐로 유입시킵니다. (제외된 파일 목록과 카운트 알림 창 제공)

### 🌿 3. 스테가노그래피 보이지 않는 서명 (Invisible Stego Seal Embedded)
- 고해상도 이미지 데이터의 파란색(Blue) 및 투명도(Alpha) 영역의 미세 공간 주파수 대역에 소유자, 고유 ID, 암호화 문자열을 소리 없이 녹여 넣습니다.
- 크롭하거나 화면을 캡쳐하더라도 데이터 훼손이 극도로 적게 유지되어, 눈에 보이는 워터마크를 악의적으로 지워낸 침해물을 사후 추적하고 저작자임을 무결히 입증합니다.

### 🔑 4. SHA-256 디지털 지문 로그 및 CSV 엑셀 대장 내보내기 (Digital Fingerprint Tracking & Export)
- 업로드된 파일의 오리지널 고유 상태값과 변환이 완료된 최종 파일의 상태값에 대한 256비트 암호화 해시를 **Web Crypto API**를 사용하여 브라우저 로컬 환경에서 군사 레벨의 안전성으로 초고속 연산합니다.
- **[Export Audit Log (CSV)]** 기능을 활용하여, 전체 세션 기록(파일명, 폴더 트리 경로, 처리 전/후 세부 해시, 적용된 워터마크 세부값, 타임스탬프)을 엑셀과 통계 프로그램 등에서 완벽히 호환되는 공식 스프레드시트로 영구 소장할 수 있습니다. (BOM UTF-8 인코딩을 적용해 한글 및 다국어가 깨지지 않고 자동 표기됩니다)

### 🧐 5. 복합 사후 검증 엔진 (Integrated Watermark Verifier)
- 도용이 의심되거나 불법 배포가 감지된 결과물 파일을 검증 탭에 넣으면 스테가노그래피 알고리즘이 동작하여, 숨겨진 암호 데이터를 즉시 추출 및 대조합니다.
- 소유권 서명의 유무, 스코어 백분율(Verification Score), 타임스탬프 기록을 대시보드 인터페이스로 즉시 판독해 줍니다.

### 📋 사전 준비 사항 (Prerequisites)

1. **컴퓨터 엔진 설치 (Node.js)**
   - 본 애플리케이션을 구동하기 위한 가상 환경인 **Node.js**가 기기에 먼저 준비되어야 합니다.
   - [Node.js 공식 다운로드 페이지(https://nodejs.org)](https://nodejs.org/)에 접속하여 가급적 **LTS(안정화 권장 버전)** 버전을 다운로드한 뒤, 일반 프로그램을 설치하듯 설치 단계를 마쳐주시기 바랍니다.

2. **최신 브라우저 사양**
   - 크로미움 보안과 내부 암호화 가속기(Crypto API)를 정교하게 탑재했기 때문에 **Google Chrome**, **Microsoft Edge**, 혹은 **Safari** 최신 버전의 브라우저에서 실행하시는 것이 가장 적합합니다.

### 💻 설치 및 가동 단계를 위한 로드맵

#### [1단계] 개발 소스코드 내려받기 (Download or Clone)
* **방법 A (가장 단순한 방법 - ZIP 다운로드)**
  1. 본 온라인 저장소(GitHub 등) 화면 우측 상단의 초록색 버튼인 `Code`를 클릭합니다.
  2. 다운로드 메뉴에서 `Download ZIP`을 클릭합니다.
  3. 다운로드된 압축 파일을 원하는 작업용 폴더에 이동시킨 후, 압축을 완전히 해제합니다.
* **방법 B (Git 명령어 도구 사용)**
  - 터미널을 실행한 뒤, 컴퓨터가 원하는 드라이브 경로로 이동하여 아래 명령어를 순서대로 입력합니다:
    ```bash
    git clone <your-repository-url>
    cd <project-directory>
    ```

#### [2단계] 로컬 터미널/콘솔 창 실행하기 (Open Terminal/Command Prompt)
* **Windows 환경:**
  - 압축이 풀린 루트 폴더(안에 `package.json` 파일이 있는 위치) 내부 빈 곳에서 `Shift` 키를 누른 채 마우스 우클릭을 한 뒤, **"여기에 파워쉘 창 열기(Open PowerShell window here)"** 혹은 **"Open in Terminal"** 메뉴를 누릅니다.
* **Mac 환경:**
  - 폴더 이름에 대고 마우스 우클릭을 클릭하여 **"폴더에서 새로운 터미널 열기"**를 선택하거나, 터미널 앱을 가동하여 압축해제 한 프로젝트 폴더 내부로 `cd` 명령어를 통해 이동해 줍니다.

- 터미널 창에 아래의 명령어를 복사하여 붙여넣고 키보드의 `Enter`키를 누릅니다:


#### [4단계] 나만의 개인 로컬 웹 시스템 가동 (Start Development Server)
- 라이브러리 조립이 완료되었다면, 터미널 창에 아래 명령어를 실행합니다:
  ```bash
  npm run dev
  ```
- 실행 즉시 터미널 콘솔에 안전 가동 로그와 함께 로컬 연결 주소가 표시됩니다:

  ```text
    ➜  Local:   http://localhost:3000/
  ```

- 이제 브라우저 창을 실행한 뒤 스마트폰이나 PC에서 주소창에 **`http://localhost:3000`**를 입력하고 접속하시면 네트워크 망이 없는 환경에서도 무관하게 완벽한 기능의 워터마크 스캐너 및 빌더를 직접 화면으로 감상하고 전 과정 가공물 처리를 완성할 수 있게 됩니다.

## 4. 사용 방법

### [1단계] 아카이빙 파일/폴더 무더기 등록
1. 상단 보호모드 메뉴에서 **"Batch Watermarker (배치 등록)"** 탭을 선택합니다.
2. PC에서 보호가 필요한 폴더 혹은 다중 파일 구조를 선택한 뒤 점선으로 채워진 **"Drag & drop Files or Folders directly here"** 영역으로 드롭합니다. 
3. 혹은 **`Select Folder`** 버튼을 클릭하여 통째로 등록할 폴더를 일괄 임포트합니다.
4. 어도비 전용 파일이 포함되었을 경우, 상단 알림 영역을 통해 스킵 목록과 사유를 편리하게 고지해 줍니다. 
5. 업로드 대기 목록 하단의 각 항목 우측에는 **`SHA256: 7f83b1c2...` (Hashing...)** 으로 개별 원문 디지털 지문이 자동 산정되어 투명하게 디스플레이됩니다.

### [2단계] 워터마크 상세 매개변수 커스터마이징
설정 패널(Watermark Settings Panel)을 통해 다음 항목들을 마우스 이동 및 슬라이더 제어로 간편하게 셋업합니다:
- **시각적 워터마크 타입 선택**: `Text` (문자열 직접 기입) 혹은 `Logo Image` (보유하고 계신 고유 브랜딩 투명 로고 PNG 이미지 파일 업로드 연계) 중 선택합니다.
- **수치 제어**: 본문 크기, 라인 자간, 오버레이 회전 각도, 타일링 격자 반복 여부와 로고 마크 배율을 섬세하게 변경합니다.
- **보이지 않는 스테고 씰(Stego) 체크**: `Embed Steganography Signature` 필터를 활성화하고 본인의 고유 서명 값(예: `© 256 Studio Inc.`)을 기입하여 법적 영구 증거를 설계합니다.

### [3단계] 배치 변환 일괄 시작
- 우측 설정 영역 최하단의 녹색 활성 버튼인 **`Apply & Export Batch`** 버튼을 클릭하여 탑재된 초정밀 변환 엔진을 대칭 실행합니다.
- 내부 프로세서가 개별 파일들의 스냅샷과 비디오 프레임을 병렬 연산하며 진행률 게이지를 채웁니다.

### [4단계] 암호 해싱 대조 및 아카이빙 보관
- 완료 영역에 새롭게 성공한 파일이 팝업되면, 각각의 디테일 프레임 하단에 **`Orig-SHA (원본 디지털 지문)`** 과 **`Result-SHA (최종 가공작 고유 해시)`** 가 쌍으로 추출되어 완벽한 고유성을 검증해 줍니다.
- **`Export Audit Log (CSV)`** 버튼을 가볍게 1회 클릭하면 자동으로 다운로드되는 엑셀 대응 표를 통해, 회사 내부 결재용이나 타 사법기관 전달용 소유권 감사 증거 데이터(Forensic Audit Roll)를 아카이빙 폴더 최상단에 안전히 저장해둘 수 있습니다.
- **`Save Structured ZIP`** 버튼을 클릭하여 본래 업로드할 때 가졌던 원형 폴더 트리를 100% 원형대로 유지한 단일 일괄 압축 ZIP 파일을 획득해냅니다.

### [5단계] 사후 소유권 도용 및 저작자 증명
- 저작권 입증이 극도로 요구될 때는 상단의 **"Verify Watermark (실시간 검증)"** 탭으로 전환합니다.
- 도용당한 정황이 있는 결과물 파일이나 원본에서 2차 무단 변형 처리된 이미지를 검증 드롭존에 투하합니다.
- **Verification Report Analyzer**가 가동되어 탑재된 서명 일치 성공 여, 시각 지표 스코어, 암호화 문자열 복호화 산출 타임스탬프를 담은 다이내믹 보고서를 리포팅해 줍니다.

## 5. Python CLI 포렌식 분석 유틸리티 사용 가이드

본 애플리케이션의 핵심 검증 기술은 백엔드 및 대용량 서버 파이프라인과의 연계를 위해 강력한 **Python CLI 모듈(`robust_watermark.py`)**로도 함께 제공됩니다. 대량의 도용 의심 정황 파일을 로컬이나 서버 터미널에서 초고속으로 일괄 분석(Batch Detection)하고 정합 대조할 때 매우 유용합니다.

### 🐍 필수 환경 사양 및 라이브러리 구성
파이썬 스크립트를 로컬 기기에서 구동하기 위해 아래 패키지 의존성들을 환경에 맞게 먼저 빌드해주어야 합니다:
```bash
pip install numpy opencv-python PyWavelets
```

### 🔍 1. 워터마크 고유 신호 감지 및 무결성 판독 (Detect)

침해나 도용이 의심되는 저작물의 주위 픽셀을 분석하여, 이미지 심층 주파수에 잠재된 암호 신호 및 복원 텍스트를 실시간 해독합니다. 자르기(Cropping), 스크린샷 캡쳐, 리사이징 등의 변형이 가해진 이미지에서도 적응형 스케일 스윙 기술을 작동시켜 강력히 검출해냅니다.

* **기본 실행 구문**
  ```bash
  python robust_watermark.py detect <분석_대상_이미지_경로>
  ```
* **실행 예시 (단일 검사)**
  ```bash
  python robust_watermark.py detect ./suspected_leak_cropped.png
  ```
* **터미널 진단 출력 리포트 예시**
  ```text
  =======================================================
          🧐 DIGITAL WATERMARK VERIFICATION REPORT       
  =======================================================
   • 검증 결과(Status)    : ✅ VERIFIED (정품 확인)
   • 무결성 매칭율(Score) : 87%
   • 소유자 정보(Owner)   : © 202 Studio LLC.
   • 진단 가이드(Remarks) : 주파수 복구 스캔 통과 (스케일 배율: 0.75x, 오프셋: 3,2px). 일치율: 87.5%. 보이지 않는 법적 저작권 씰이 유지되었습니다.
  =======================================================
  ```

### 📊 2. 폴더 내 모든 가공물 일괄 고속 분석 (Batch-Detect)

폴더 트리에 무작위로 분산 저장된 원화나 수십 장의 다운로드 대상을 한 번에 검출하여, 대치 정합율을 일람표 형식으로 고속 출력합니다. 법정 아카이빙 대장 작성 및 불법 복제 필터링 시스템 구축에 유용합니다.

* **기본 실행 구문**
  ```bash
  python robust_watermark.py batch-detect <검사_대상_폴더_경로>
  ```
* **실행 예시 (대용량 일괄 스캔)**
  ```bash
  python robust_watermark.py batch-detect ./target_leak_folder
  ```
* **터미널 콘솔 일람표 출력 예시**
  ```text
  🔍 폴더 내 모든 파일 일괄 스캔(Batch-Detection)을 개시합니다... (총 3개 파일)
   - 대상 폴더: ./target_leak_folder
  --------------------------------------------------------------------------------
  파일명 (Filename)               | 결과 (Status) | 매칭율 (Score) | 식별된 소유자 (Owner)
  --------------------------------------------------------------------------------
  watermarked_main.png           | ✅ VERIFIED   | 100%           | © 2026 Core Tech
  leak_cropped_shot.png          | ✅ VERIFIED   | 83%            | © 2026 Core Tech
  normal_unprotected_etc.jpg     | ❌ FAILED     | 37%            | -
  --------------------------------------------------------------------------------
  📊 스캔 총평: 검증 이미지 2개 | 미매칭/원본 1개 (총 3개 중)
  ================================================================================
  ```

### 🛡️ 3. 대량 이미지 무가시성 일괄 인베딩 (Batch-Embed)

본 애플리케이션 프런트엔드의 다중 빌더와 동일한 논리로, 특정 폴더 내의 다수 미디어 자료 하부에 무가시성 주주파수(DWT-SS) 스테고 사인을 대용량 병렬 주입하여 정형화 저장합니다.

* **기본 실행 구문**
  ```bash
  python robust_watermark.py batch-embed <원본_입력_폴더> <가공_출력_폴더> "<소유권_서명_텍스트>"
  ```
* **실행 예시 (일괄 무가시 서명 장치)**
  ```bash
  python robust_watermark.py batch-embed ./originals_dir ./packaged_outputs "© 2026 Author Master"
  ```

### 💻 4. 단일 타겟 무가시 인베딩 단독 가동 (Embed)

단일 고용량 자산 파일에 단독으로 무가시성 DWT 주파수 서명을 봉인합니다.

* **기본 실행 구문**
  ```bash
  python robust_watermark.py embed <원본_이미지> <출력_이미지> "<소유권_서명_텍스트>"
  ```
* **실행 예시**
  ```bash
  python robust_watermark.py embed original.png secured_hybrid.png "© 2026 Author Master"
  ```
